a teg의 속성

밴드홈
  1. a(anchor 배의 닷) - 테그에 링크를 걸겠다는 명령어
  2. href - 클릭하면 링크되는 주소
  3. target - 위치 ,_blank 새로운페이지에
  4. title - 마우스를 테그에 놓으면 표시되는 글자
  • 속성의 순서는 중요하지 않다
  • ol 은 order list로 순서가 있는 group list
  • ul 은 unorder list로 순서가 없는 group list
  • li 는 list item 으로 줄바꿈이 된다

개발도구 : atom, bracket

emmit의 기능

  • !+tab : html의 기본 list
  • div+p+li : 형제관계
  • nav>ul>li>a
  • nav>ul>(li>a)*5 : group
  • nav#idname>ul>li : #idname)
  • li.classname>a : .classname
  • a[target="_blank" title="emmit"] + tab
  • li>a>{text} +tab : {}는 text
  • li.item$*5 : item1~5까지

  • 특별한 기능은 없지만 의미를 가지고있다.
  • header : 이 웹의 가장 중심이 되는 코딩부분을 header로 감싸준다
  • nav : 이 웹에서 네이게이션의 기능을 가진 부분에 명시해준다

  • article: 본문에 해당하는 부분을 명시해준다
  • footer : 이 웹의 가장 아래에 깔리는 코딩부분을 footer로 감싸준다

    모바일 화면

  • 모바일지원(viewport) : 마우스 오른쪽>검사>toggle device toolbar(elements옆 네모) 누르면 왼쪽에 화면나옴> responsive 누르면 휴대폰 마다 보여지는 화면 볼수있다

    • 모바일 화면폭 맞추기

  • head >
  • meta name="viewport" content="width=device-width,initial-scale=1.0">
  • 화면의 폭을 장치의 폭만큼으로 해라 하는 명령어.
  • 모바일 마다 화면의 폭이 다르기 때문에 이 명령어를 사용하면 편리하다.
  • viewport : 화면
  • width=device-width: 화면의 폭을 장치의 폭과같이해라.
  • initial-scale=1.0:줌을 1.0으로 하라=확대,축소없이

    • network

  • 마우스오른쪽>검사>network : 웹브라우저와 서버가 데이터를 주고받는 상황
